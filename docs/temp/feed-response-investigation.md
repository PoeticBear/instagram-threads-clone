# /post/feed 原始响应排查样本

- 生成时间: 2026-06-25 09:37:33
- 环境: dev (http://192.168.1.27:8005/)
- 接口: `GET /post/feed?page=1&size=50`
- 鉴权: `Authorization: Bearer <access_token>` (用户 14dev, id=1000382)
- 抓取条数: 9 (后端 total=9)

---

## 一、关键发现(先看这里)

### 1. feed 嵌套的 `quote_post` 全部 `media_list=[]`

dev 环境的 9 条 feed 中有 4 条是引用帖,它们的 `quote_post.media_list` **字段存在但恒为空数组**:

| 主帖 ID | 引用帖 ID | feed 中 `quote_post.media_list` |
| --- | --- | --- |
| 2000166 | 2000164 | `[]` |
| 2000159 | 2000158 | `[]` |
| 2000133 | 2000132 | `[]` |
| 2000115 | 2000114 | `[]` |

### 2. fallback 接口 `/post/detail/{id}` 能补齐媒体

| 引用帖 ID | `/post/detail/{id}` 中 `media_list` | 媒体类型 |
| --- | --- | --- |
| **2000164** | **`[1 条]`** | **`media_type=2(视频)`** ✓ |
| 2000158 | `[]` | — |
| 2000132 | `[]` | — |
| 2000114 | `[]` | — |

**结论**:`/post/feed` 嵌的 `quote_post` 是后端**主动裁掉**的精简版(只留元信息、不带媒体);
客户端要拿到被引用帖的媒体,必须**走 `/post/detail/{quote_post_id}` 二次拉取**。

**Bug 定位方向**:`/post/detail/2000164` 返回的视频数据是齐的,所以问题大概率在
**客户端 fallback 触发后到渲染之间的链路**(`feedpost.dart` 的 `_maybeFetchQuotePost` / 
`_effectiveQuotePost` / `_buildQuoteCard` / `_buildQuoteVideoPoster`)。
