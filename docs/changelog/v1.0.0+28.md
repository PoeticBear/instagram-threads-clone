# v1.0.0+28

> 前一构建：v1.0.0+27
> 状态：🟢 当前版本

## ✨ 新增功能
- **点击对焦 + 同步设置曝光点**：相机预览中点击一次即可同时设置对焦点与曝光点；前置镜头自动对 x 坐标做镜像；2 秒后对焦框自动消失（OpenSpec `enhance-compose-camera`）
- **曝光补偿滑杆**：基于设备 `getMin/MaxExposureOffset` 自动确定范围与步长，垂直滑杆实时调整；镜头切换后自动 clamp 到新范围（OpenSpec `enhance-compose-camera`）
- **九宫格辅助线**：4 条辅助线 overlay，开关持久化到 `SharedPreferences`，不影响对焦与缩放手势（OpenSpec `enhance-compose-camera`）
- **3 秒倒计时**：照片 / 视频模式均支持，倒计时期间切镜头、切模式、关闭页面、App 进入后台都会自动取消（OpenSpec `enhance-compose-camera`）
- **物理镜头切换**：根据设备实际枚举到的 `lensType` 显示 0.5× / 1× / 2× 入口（缺啥不显示啥），切换会短暂 loading；录像中拒绝切换（OpenSpec `enhance-compose-camera`）
- **受控画质档位**：仅 720p/30fps 与 1080p/30fps 两个档位，默认 1080p/30fps；偏好持久化（OpenSpec `enhance-compose-camera`）
- **多张照片会话拍摄**：相机页不再拍完一张就退出，可在会话内连续拍摄至发布页剩余配额；顶部显示已拍数量与缩略图，点击「完成」批量返回；视频模式仍按单次返回（OpenSpec `enhance-compose-camera`）
- **拍后静态图片滤镜**：5 项基础滤镜（原图 / 黑白 / 暖色 / 冷色 / 高对比度），使用 `image` 包做静态处理；处理超时或失败时显示错误并允许继续使用原图（OpenSpec `enhance-compose-camera`）

## 🐛 修复
- **录制中进入后台 / 关闭页面不再丢视频**：`didChangeAppLifecycleState.inactive` 与关闭按钮都会先调用 `stopVideoRecording()` 走正常停止流程，再 dispose 控制器；防止未完成视频被直接丢弃（OpenSpec `enhance-compose-camera`）
- **视频缩略图失败不再把视频文件当图片写入**：失败时 `thumbnail` 字段为 `null`，UI 使用统一的占位兜底，避免 `Image.file` 尝试解码 mp4 文件出错（OpenSpec `enhance-compose-camera`）
- **视频时长不一致**：UI 计时上限 / 录制逻辑 / 上传校验 / 错误文案 / `VideoProcessor` 默认值全部统一为 300 秒（5 分钟）；删除了「60 秒」相关残留（OpenSpec `enhance-compose-camera`）
- **缩放基准累乘误差**：双指缩放改为 `onScaleStart` 保存基准 zoom，避免长手势过程中的累乘跳变（OpenSpec `enhance-compose-camera`）

## ♻️ 重构
- **控制器重建串行化**：引入 `_pendingGeneration / _myGeneration` 防止旧 controller 异步回调覆盖新 controller 状态；任何 `_startCamera` 入口（模式切换 / 镜头切换 / 画质切换 / 生命周期恢复）走同一条串行路径（OpenSpec `enhance-compose-camera`）
- **相机拍摄结果接入统一校验**：抽出 `CameraResultValidator`，图片 10MB / 视频 100MB / 视频时长 300 秒；相册与相机共用同一套校验（OpenSpec `enhance-compose-camera`）
- **新增 `CameraLensHelper` / `CameraQualityPreset` / `ComposeCameraConfirmPage` / `CameraResultValidator`** 四个新文件，相机页职责更聚焦（OpenSpec `enhance-compose-camera`）
