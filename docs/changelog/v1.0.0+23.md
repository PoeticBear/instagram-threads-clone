# v1.0.0+23

> 发布日期：2026-07-16
> 前一构建：v1.0.0+22
> 状态：🟢 已发布（TestFlight 上传成功 `** EXPORT SUCCEEDED **`，等待 Apple 端处理 5–15 分钟）

> 本版核心主题：**「写文字」流程改为两步交接 + 顺手修一条 feedpost 编辑卡住的旧 bug** —— 用户在「写文字」页敲完文字 + 选好渐变样式后，点「确认」截图卡片，自动跳到普通发文页接管卡片 PNG；用户可继续加图 / 改文 / 加投票 / 定时，再点右上「发布」提交。

## ✨ 新增功能

- **「写文字」AppBar「发布」→「确认」，截图卡片交接普通发文页**（`ea305b0`）— `TextNotePage` 不再闭环发 post；`Navigator.pushReplacement` 把 `(text, imageDraft)` 作为 `ComposePost.initialContent` + `initialMediaDrafts` 透传，由 `ComposePost` 接管后续编辑与最终发布。栈结构从 `[Home, TextNotePage, ...]` 收敛为 `[Home, ComposePost]`。
- **「写文字」放行空文字（纯渐变卡片合法）**（`ea305b0`）— 按钮可用条件由 `_hasContent` 放宽为 `!_isConfirming`，允许只发一张渐变卡（不需要敲任何文字即可点确认进入）。
- **`ComposePost` 新增可选入参 `initialMediaDrafts`**（`ea305b0`）— 编辑模式下不生效（继续走原帖 media 恢复链路），仅在 `editingPostId == null` 时预填 `_mediaDrafts`。`TextNotePage` 直接利用此入参透传卡片 PNG。

## 🐛 修复

- **编辑帖子后卡在已清空的发布页**（`e9408f1`）— Feed 长按自己的帖选「编辑」→ 改文字 → 点「发布」，原 pre-existing bug：`ComposePost._submit` 成功路径调 `widget.onPostSuccess?.call()` 缺省为 no-op，导致页面不自我 pop。补 `MaterialPageRoute.builder` 闭包内的 `routeContext` 捕获，传 `onPostSuccess: () => Navigator.of(routeContext).pop()` 与 `onCancel: () => Navigator.of(routeContext).pop()` 两条对称回调。
- **写文字页确认后同样卡住**（`ea305b0`，与上一条同根因）— `change-text-note-handoff` 把 `TextNotePage` 的 `_publish` 整体替换为 `pushReplacement` 后，没补 `onPostSuccess` 回调就会触发同 bug，已在上方「新增功能」同 commit 里一并修掉。
- **定时发布 picker 默认时间归整到整点**（`65031e6`）— 避免继承秒/微秒导致 picker 一打开就和下一次整点冲突；本次随版本提交一并上线。
