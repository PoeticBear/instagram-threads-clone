- 帖子功能模块服务端API文档：openapi_docs/post.json

  1. 提及身份绑定 — 用 userId 记录被提及用户,随帖子提交服务端(当前 @username 只是纯文本,改名即失效)。
  2. 正文 @username 可点击 — 已发布帖子里点击 @ 跳转该用户主页(走 userId,不是用户名)。
  3. 通知被提及的用户 — 帖子发布后,被 @ 的用户收到"xx 提及了你"的通知(否则 @ 功能毫无业务价值)。
  4. 选中后高亮显示 — 选中用户后,正文里的 @username 以主题色高亮,与其他文字区分。
  5. 回复/引用场景覆盖 — 评论框、转发引用框的 @ 能力应与主发帖页保持一致,目前仅在发帖页实现。

> 本期另含：WebSocket 实时通知接入、登录后强制补填用户名、视频查看页方向分流、引用帖视频渲染修复、回复点赞乐观更新等（详见下方分类）。

## ✨ 新增功能
- **@提及完整上线**：发帖正文里 @ 用户现在绑定真实身份（用 userId 记录），对方改名也不会让旧 @ 失效；发布后正文中的 @username 可点击跳转该用户主页，并以主题色高亮区分；回复场景（评论框、引用框）也补齐同样的 @ 能力，与主发帖页一致（`a5492dc`、`a1c2141`、`22a5c99`）
- **实时通知（WebSocket）接入**：接入 WebSocket 推送骨架与全部通知事件（点赞 / 回复 / 转发 / 引用 / @提及 等），相关互动可实时触达，不再依赖手动下拉刷新（`ada537a`、`096928c`）
- **登录后强制补填用户名**：新用户登录后弹出不可关闭的弹窗强制补填 username，走专用 `/user/username` 接口，避免后续功能因缺少用户名而异常（`936df05`）
- **视频查看页方向分流**：竖屏视频自动沉浸铺满全屏；横屏视频可手动切换横屏播放，画面比例更贴合内容（`ad95d42`）

## 🐛 修复
- **回复点赞即时反馈**：回复点赞改为乐观更新——点一下心形立即翻转，接口失败再回滚，告别点击无反应的延迟感（`fc02d7c`）
- **引用帖视频/图片渲染**：修复引用帖中视频和图片不显示的问题，补齐 poster 三态（加载中 / 成功 / 失败）与媒体懒加载兜底（`5b7c9d6`）
- **引用帖不存在兜底**：被引用的原帖已被删除时不再显示空白或报错，改为本地化兜底文案，同时压住冗余的全局错误提示（`4ebc2b1`）
- **@提及服务端契约对齐**：适配 `mentioned_users_info` + 逗号分隔 userId 的服务端契约，确保提及身份正确同步（`22a5c99`）
- **WS 鉴权与路径对齐**：对齐开发环境 WS 路径与服务端鉴权约定（query 带 `access_token`），修复 mention 同步问题并增强排障日志（`dfa10cd`、`9688eea`）
