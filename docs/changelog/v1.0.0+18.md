## ✨ 新增功能
- 个人中心 Threads Tab 支持下拉刷新：在帖子列表顶部下拉即可重新拉取该用户的帖子，无需切走再切回。原本下拉是「死」的手势，现在与首页 Feed、通知、关注列表的下拉行为保持一致
- 个人中心 Media Tab 支持下拉刷新：媒体网格同样支持下拉重新加载；Media 是帖子列表的派生视图，所以一次下拉会同时刷新两个 Tab，无需重复操作
- 个人中心 Reposts Tab 补齐下拉手势：原本 Reposts Tab 完全是占位文案（下拉无反应），现在加上 RefreshIndicator 让三个 Tab 的下拉体验保持一致；但因当前没有「某用户转发列表」接口，onRefresh 暂为 no-op，UI 仍展示「暂无转发」等待后续接口上线

## 🐛 修复
- 修复引用卡头像长图（竖屏 / 横图）变形：之前 `CachedNetworkImage` 缺 `fit` 参数，默认 `BoxFit.fill` 会把图片横向拉伸成"胖"圆形。改为 `BoxFit.cover` 后等比缩放居中裁切，与主帖头像（`UserAvatarWithFollow`）行为一致
- 顺带修复帖子回复列表头像的同源 Bug：`_buildAvatar` 内 `CachedNetworkImage` 同样缺 `fit`，现统一改为 `BoxFit.cover`（涉及 `feedpost.dart:158` 与 `reply_bottom_sheet.dart:383`）
- 增强引用卡 / 回复列表头像的加载与失败态：之前无 `placeholder` / `errorWidget`，加载中或失败时头像区域为空白。现补齐两个回调——加载中显示居中 spinner，失败时降级为圆形默认头像（`Icons.person`），体验与首页快捷发帖区（`feed.dart`）一致
- 修复引用帖子发布后被引用区域不显示：`PostState.createPost` 在把服务端返回的新帖插入到 `_feedlist` 时走的是一份精简手写构造，漏拷 `quoteRepostId` 与 `quotePost` 两个字段。导致 `FeedPostWidget` 的两道防线双双失效——嵌套数据为空、且 `_maybeFetchQuotePost` 因 `quoteRepostId` 也为空直接 return 不触发兜底拉取——必须下拉刷新 Feed 才能看到被引用区域。改为复用既有的 `_apiPostToModel(post)`，与服务端响应字段自动对齐，并顺带覆盖了此前手工构造同样漏掉的 `isPinned` / `editCount` / `isSensitive` 等字段
