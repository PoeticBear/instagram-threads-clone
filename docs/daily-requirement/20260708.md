# 2026-07-08 需求背景：登录方式对接

> **说明**：以下三个接口为接下来客户端需求开发的**背景信息**，均提取自服务端 OpenAPI 文档 `openapi_docs/versions/openapi_20260708.json`（由 `scripts/openapi_doc.py` 截取）。三者共同构成「**第三方登录 + 手机号登录**」能力的服务端契约，是后续客户端登录页改造的依据。

## 接口清单

| # | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 1 | POST | `/auth/google/login` | Google 账号登录（前端传授权码，后端向 Google OAuth 验证） |
| 2 | POST | `/auth/sms/send` | 发送短信验证码 |
| 3 | POST | `/auth/sms/signin` | 手机号 + 验证码登录 / 自动注册 |

## 通用约定（三个接口共用）

**统一响应体 `ResponseModel[T]`**：所有接口 200 返回都包裹在该结构里。

```json
{
  "code": 0,          // 业务码，整数
  "msg": "string",    // 业务提示文案
  "data": T | null    // 业务数据，失败或无数据时为 null
}
```

**422 校验错误**：三个接口的请求体校验失败时统一返回 FastAPI 标准 `HTTPValidationError`（`detail: [{loc, msg, type}]`），不再在每个接口下重复。

**可选请求头**（`google/login` 与 `sms/signin` 用于设备识别）：

| Header | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `user-agent` | string | ❌ | 用户代理 |
| `device-os` | string | ❌ | 系统类型 |
| `device-name` | string | ❌ | 设备名称 |

## 1. POST `/auth/google/login` — Google 登录

- **Tags**：认证
- **Summary**：Google 登录
- **Description**：Google 账号登录。前端传入 Google 授权码，后端向 Google OAuth 服务器验证并换取用户标识；**已有用户直接登录，新用户自动注册并绑定 Google 账号**。
- **OperationId**：`google_login_auth_google_login_post`
- **请求头**：见上方「可选请求头」。

**请求体 `GoogleLoginRequest`**（`application/json`）：

| 字段 | 类型 | 必填 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| `code` | string | ✅ | minLength 1 | Google 登录授权码 |

**响应 200 `data` → `SigninResponse`**（登录成功应答对象）：

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | integer | > 0 | 用户ID |
| `username` | string | maxLength 32 | 用户名 |
| `avatar` | string | maxLength 512 | 头像 |
| `access_token` | string | — | 访问令牌 |
| `refresh_token` | string | — | 刷新令牌 |
| `display_name` | string | — | 显示名称 |

---

## 2. POST `/auth/sms/send` — 发送短信验证码

- **Tags**：认证
- **Summary**：发送短信验证码
- **Description**：发送短信验证码
- **OperationId**：`sms_send_auth_sms_send_post`

**请求体 `SendSMSRequest`**（`application/json`）：

| 字段 | 类型 | 必填 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| `phone_country_code` | string | ✅ | 2–10 | 国际区号，如 `+86` |
| `phone` | string | ✅ | 1–20 | 手机号 |

**响应 200 `data` → `OKResponse`**：空对象 `{}`，发送结果以顶层 `code` / `msg` 表达。

---

## 3. POST `/auth/sms/signin` — 短信验证码登录 / 注册

- **Tags**：认证
- **Summary**：短信验证码登录
- **Description**：短信验证码登录 / 注册。手机号 + 验证码，**已注册用户直接登录，新用户自动注册**。
- **OperationId**：`sms_signin_auth_sms_signin_post`
- **请求头**：见上方「可选请求头」。

**请求体 `SMSLoginRequest`**（`application/json`）：

| 字段 | 类型 | 必填 | 约束 | 说明 |
| --- | --- | --- | --- | --- |
| `phone_country_code` | string | ✅ | 2–10 | 国际区号，如 `+86` |
| `phone` | string | ✅ | 1–20 | 手机号 |
| `code` | string | ✅ | 4–6 | 短信验证码 |

**响应 200 `data` → `SigninResponse`**：结构同接口 1（`id` / `username` / `avatar` / `access_token` / `refresh_token` / `display_name`）。

---

## 备注

- 接口 1 与接口 3 成功后返回的 `SigninResponse` 完全一致 —— 客户端登录态落地逻辑（存 `access_token` / `refresh_token`、跳转主页）可复用同一套处理。
- 接口 2 与接口 3 配合使用：先 `sms/send` 发码，再 `sms/signin` 用 `phone_country_code + phone + code` 换取登录态。
- 业务失败（如验证码错误、手机号未授权）通过顶层 `code` / `msg` 区分，`data` 为 `null`；客户端需以 `code` 判定结果，不要仅依赖 HTTP 状态码。
